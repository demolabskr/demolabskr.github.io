# 실습 3

### 1. 평가문항

"문제 유형2. 아래 리뷰 데이터 6건을 분석해, 개선 우선순위와 실행안을 도출하고 이를 반영한 AI 프롬프트를 설계해 보세요.

<리뷰 데이터>
1) "강의 다운로드가 자꾸 실패해요. 와이파이에서도 80%에서 멈춥니다."
2) "진도율이 초기화됐어요. 지난주 기록이 사라졌습니다."
3) "결제는 됐는데 프리미엄 강의가 잠겨있습니다."
4) "알림이 너무 많아요. 끌 수 있는 메뉴를 못 찾겠습니다."
5) "자막이 늦게 나와서 따라가기 힘듭니다."
6) "고객센터 답변이 3일이나 걸렸어요."

<제출할 결과물>
1) 이슈 분류표 1개
- 카테고리 / 근거 인용 / 빈도(건수) / 심각도(상·중·하) 포함

2) 개선 우선순위 3개
- 선택 근거(영향도/신뢰도/해결 난이도) 1줄씩

3) 개선 실행안 3개
- 기획/UX/운영 중 최소 2개 영역 포함
- 각 실행안마다 성공지표 1개 포함

4) 프롬프트 2종(각 8~12줄 내)
(A) 리뷰 요약·분류 프롬프트
(B) 리뷰 답변·재발방지 제안 프롬프트
※ 2종 프롬프트 전체에서 '역할/단계/형식/예시' 중 최소 2가지는 꼭 사용해요.

5) 입력 줄이기 규칙 2개(토큰/문맥 창 관점)

6) 적용 예시 1건(A->B 결과)
- 위 리뷰 중 1건을 골라 A 결과와 B 결과를 순서대로 보여 주세요.

<제출형식> AI 활용 결과물은 파일 형태로 제출할 수 있으며, 여러 파일을 제출하는 경우 하나의 압축 파일로 제출해 주세요."

### 정답 및 해설

"1) 이슈 분류표 1개 — 정답 예시

| 카테고리 | 근거 인용 | 빈도 | 심각도 |
| --- | --- | --- | --- |
| 다운로드/재생 오류 | "80%에서 멈춥니다" | 1 | 중 |
| 학습 기록/동기화 | "진도율이 초기화" | 1 | 상 |
| 결제/권한 동기화 | "결제는 됐는데 잠겨" | 1 | 상 |
| 알림/설정 UX | "알림이 너무 많아요" | 1 | 중 |
| 접근성(자막) | "자막이 늦게 나와" | 1 | 중 |
| 고객지원 응답 | "답변이 3일" | 1 | 중 |

2) 개선 우선순위 3개 — 정답 예시

1. 결제/권한 동기화: 영향도 높고 신뢰도에 직결, 해결 난이도는 중.
2. 학습 기록/동기화: 기록 손실은 이탈 위험이 커서 우선 처리 필요.
3. 다운로드/재생 오류: 사용 빈도가 높아 체감 불편이 큼.

3) 개선 실행안 3개 — 정답 예시

1. 결제-권한 동기화 자동 복구 버튼 추가(기획) / 성공지표: 관련 문의 30% 감소
2. 학습 기록 서버 백업 주기 명시 + 복구 안내 배너(UX) / 성공지표: 기록 복구 요청 대비 해결율 90%
3. 다운로드 실패 시 에러 코드 노출 + 재시도 흐름 단순화(운영/UX) / 성공지표: 다운로드 실패율 20% 감소

4) 프롬프트 2종(각 8~12줄 내)

(A) 리뷰 요약·분류 프롬프트 — 정답 예시(9줄)

1. 역할: 너는 "리뷰 분석 담당자"야.
2. 입력: {리뷰텍스트}
3. 목표: 핵심 불만을 2줄 요약하고 카테고리 1개로 분류해.
4. 단계1: 불만 요약 2줄 작성.
5. 단계2: 카테고리 1개와 심각도(상/중/하) 선택.
6. 추측 금지, 리뷰에 있는 표현만 사용해.
7. 출력 형식 고정:
8. [요약] 1줄 / 2줄
9. [분류] 카테고리, 심각도

(B) 리뷰 답변·재발방지 제안 프롬프트 — 정답 예시(10줄)

1. 역할: 너는 "앱 고객응대 담당자"야.
2. 기준: 공손·간결, 확정표현 금지.
3. 입력: {A_요약결과}
4. 목표: 답변 + 재발방지 제안 1개 + 다음 행동 1개 작성.
5. 단계1: 공감 1문장 + 현재 조치 안내.
6. 단계2: 재발방지 제안 1개(제품 개선 관점).
7. 단계3: 고객이 제공할 최소 정보 1개 요청.
8. 출력 형식 고정:
9. [답변] / [재발방지 제안] / [다음 행동]
10. 개인정보는 마스킹 원칙을 지켜.

5) 입력 줄이기 규칙 2개(토큰/문맥 창 관점) — 정답 예시

1. 리뷰가 길면 "불만 1줄 + 상황 1줄"로 재작성해 넣는다.
2. 동일 주제 리뷰는 대표 1개만 넣고 나머지는 키워드 목록으로 축약한다.

6) 적용 예시 1건(A->B 결과) — 정답 예시

선택 리뷰
"결제는 됐는데 프리미엄 강의가 잠겨있습니다."

A 결과
* [요약] 결제 완료 후 프리미엄 강의 접근 불가 / 권한이 잠긴 상태
* [분류] 결제/권한 동기화, 상

B 결과
[답변] 불편을 드려 죄송합니다. 결제 후 권한 동기화가 지연될 수 있어 확인 후 안내드리겠습니다.
[재발방지 제안] 결제 완료 즉시 권한 상태를 재동기화하는 자동 복구 절차를 적용하겠습니다.
[다음 행동] 결제된 날짜를 알려주시면 확인 후 안내드리겠습니다."

### 채점 기준

"생성형 AI 활용 과제 특성상 학습자별 산출물이 다양하게 나타날 수 있습니다. '정답 및 해설'은 참고용 예시로 활용하며, 평가는 평가문항의 제출 조건 및 채점기준에 따라 배점·채점합니다.

1) 이슈 분류표 1개 (20점)
* 체크포인트(각 5점, 총 20점)
  * 1-1 카테고리 분류가 명확
  * 1-2 근거 인용이 포함
  * 1-3 빈도/심각도 표기
  * 1-4 표 형식 준수
* 감점 트리거: 근거 없는 추측 카테고리 추가

2) 개선 우선순위 3개 (20점)
* 체크포인트(각 5점)
  * 2-1 우선순위 3개 제시
  * 2-2 영향도 근거 명시
  * 2-3 신뢰도 또는 난이도 근거 명시
  * 2-4 데이터와 연결된 우선순위

3) 개선 실행안 3개 (20점)
* 체크포인트(각 5점)
  * 3-1 기획/UX/운영 중 2개 이상 영역 포함
  * 3-2 실행안이 구체적
  * 3-3 성공지표 포함
  * 3-4 우선순위와 연계됨

4) 프롬프트 2종(각 8~12줄 내) (25점)
* 체크포인트(각 5점)
  * 4-1 A/B 구성·형식 충족
  * 4-2 줄수 준수
  * 4-3 역할 포함
  * 4-4 단계/형식/예시 중 1개 이상 포함
  * 4-5 출력 통제가 작동함
* 감점 트리거: 프롬프트가 아니라 결과문만 작성

5) 입력 줄이기 규칙 2개(토큰/문맥 창 관점) (5점)
* 체크포인트(각 2.5점)
  * 5-1 실행 가능한 규칙 1개
  * 5-2 실행 가능한 규칙 1개(위와 다른 방식)

6) 적용 예시 1건(A->B 결과) (10점)
* 체크포인트(각 5점)
  * 6-1 A 결과가 B에 반영됨
  * 6-2 답변/재발방지/다음 행동 구성 준수"

### 문항 출제 목표

데이터를 분석해 문제를 정의하고, 개선안을 도출한 뒤 이를 AI 프롬프트로 구조화하여 재사용 가능한 업무 지시서로 전환할 수 있다.
