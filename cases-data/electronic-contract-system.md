## 배경
계약 업무는 처리 속도보다 분쟁 시점의 검증 가능성이 더 중요했습니다.
초기에는 업무 담당자마다 승인 경로가 달라 같은 계약도 이력이 다르게 남았습니다.

## 실행
- 계약 단계별 승인 상태를 하나의 타임라인으로 통합했습니다.
- 전자서명 완료 이벤트를 기준으로 보관 정책이 자동 적용되게 했습니다.
- 감사 요청이 들어오면 로그를 즉시 재현할 수 있도록 조회 화면을 분리했습니다.

## 운영 포인트
기능을 먼저 늘리기보다, 추적 가능한 흐름을 먼저 고정한 뒤 기능을 붙여야 운영 리스크를 줄일 수 있습니다.
